## Goal

It should be possible to edit a card, with visual guidance. It should look like the view dialog but be editable. Needed for the TUI

## Acceptance Criteria

- [x] In-TUI edit dialog mirrors the view (`TaskDetail`) layout
- [x] All task fields editable: title, status, priority, assignee, tags, depends_on, body sections
- [x] Tab/Shift-Tab cycles fields, Enter activates edit, Esc exits field/dialog
- [x] `e` opens the in-TUI editor, `E` still launches `$EDITOR`
- [x] Save persists via `updateTask`; cancel reverts

## Notes

- New component `packages/cli/src/tui/TaskEditor.tsx` (mirrors `TaskDetail` styling with focus markers `▶ / ▷`).
- Visual guidance: focused row uses accent colour + `▶`, active edit uses `▷`; cursor block shown in multiline section editor.
- Field types:
  - text: `title`, `assignee` (single-line via `ink-text-input`)
  - tokens: `tags`, `depends_on` (comma/space-separated input, parsed on save)
  - select: `status` (config statuses + archived), `priority` (—/low/medium/high) — ↑/↓ cycles
  - multiline: every body section (Goal, Acceptance Criteria, Notes, Progress)
- Footer hints adapt by mode (browse vs editing vs section vs select).
- Save persists via `updateTask` from `@frehilm/ordna-core`; cancel just unmounts the draft.
- App-level wiring in `App.tsx`: new `Mode` `edit` with `returnTo: "browse" | "detail"` so saving from the detail dialog returns to the detail view. Board/detail bind `e` → in-TUI editor, `E` → `$EDITOR`.
