## Goal

Import / Export tasks to JIRA and Linear.

This is the **hybrid path** flagged in T-018: files remain canonical and a sync command pulls from / pushes to a remote tracker on demand. Distinct from T-008 / T-009, which make the remote tracker the live backend.

## Acceptance Criteria

- [ ] Import JIRA — pulls existing issues into `tasks/` as markdown files (one-way snapshot)
- [ ] Import Linear — same, for Linear issues
- [ ] Export JIRA — pushes `tasks/` contents back to a JIRA project (creates/updates issues)
- [ ] Export Linear — same, for Linear
- [ ] Conflict policy documented (last-write-wins vs merge-warn vs skip-newer)
- [ ] Mapping rules reuse the schema-translation logic from T-018 (no duplicate field-mapping code)

## Notes

Defer until **T-018 ships**. The provider interface that T-018 introduces gives us the field-mapping helpers and read-only schema translators that this task needs. Building sync first would mean writing that mapping twice.

Two-way sync is out of scope for v1: explicit one-shot `ordna import jira ...` / `ordna export jira ...` commands only. If users want continuous sync, they should use the live-backend mode (T-008 / T-009) instead.
