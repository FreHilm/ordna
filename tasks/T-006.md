## Goal

Find the best solution on how to implement the external integration with JIRA/LINEAR. Should it be integrated into core, or each UI, or separate?

## Acceptance Criteria

- [x] Decision made: implement as a **plugin / provider architecture** in `@frehilm/ordna-core`
- [x] File logic stays as the default (`FileTaskProvider`); remote backends are separate packages (`@frehilm/ordna-jira`, `@frehilm/ordna-linear`)
- [x] UIs (TUI / Web) consume a `TaskProvider` interface and remain backend-agnostic
- [x] Outcome documented in **T-018** with full architecture, staged rollout plan, schema-mismatch analysis, and open positioning questions

## Notes

The research outcome is the plan in T-018. Phase 1 of that plan (the core refactor to introduce `TaskProvider`) is the next concrete deliverable. Phase 2+ (actual Jira / Linear plugins) is captured by T-008 and T-009, both of which now depend on T-018.

Key decision: configuration picks one provider per repo for v1. Multi-board (file + remote in the same sidebar) is post-v1 only if demand exists.
