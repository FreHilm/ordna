## Goal

In the create task dialog TUI. Pressing esc is not working. Should close the dialog and go back to the board.

## Acceptance Criteria

## Notes

Root cause: `packages/cli/src/tui/TextPrompt.tsx` advertised "Esc cancel" but never wired ESC to `onCancel`. `ink-text-input` consumes key events but ignores escape, so the dialog had no way to close. The sibling `SelectPrompt` already handled this with its own `useInput` listener.

## Progress

- Identified bug in `TextPrompt.tsx`: no ESC handler despite the UI hint promising one. Affects create / assign / search dialogs (all use `TextPrompt`).
- Added a `useInput` listener in `TextPrompt` that calls `onCancel()` on `key.escape`.
- `pnpm -r build` clean; `pnpm -r test` all 37 tests pass.
